# 输入输出(IO)

输入输出（I/O）负责计算机和外部设备之间的数据交换。操作系统通过设备驱动、中断、DMA、缓冲和统一接口，把复杂多样的硬件包装成应用程序可以使用的抽象。

## 目标

- 完成 I/O 设备与内存之间的数据交换。
- 给应用程序提供统一、方便、尽量与设备无关的接口。
- 让 CPU 和 I/O 设备并行工作，提高系统整体利用率。
- 管理设备共享、错误处理、缓冲和调度。

## I/O 设备分类

- 块设备：以固定大小的数据块为单位读写，可以随机访问，例如[[磁盘]]、SSD。
- 字符设备：以字符流或字节流为单位收发，通常不支持随机定位，例如键盘、鼠标、串口。
- 特殊设备：时钟、显示器、网卡等，不完全符合块设备或字符设备模型。
## I/O 编址方式

- I/O 独立编址：I/O 端口地址空间与内存地址空间分开，CPU 使用专门的 I/O 指令访问设备端口。
- 内存映像编址：把设备寄存器映射到内存地址空间，CPU 像访问内存一样访问设备寄存器。

内存映像 I/O 的好处是可以复用普通 load/store 指令；I/O 独立编址的好处是设备地址空间和内存地址空间界限清楚。

## 三种基本 I/O 方式

| 方式 | CPU 参与程度 | 基本思想 | 适合场景 |
| --- | --- | --- | --- |
| [[程序控制IO]] | 高 | CPU 反复轮询设备状态并搬运数据 | 简单设备、小数据量 |
| [[中断驱动IO]] | 中 | 设备完成后用中断通知 CPU | 低速设备、异步事件 |
| [[DMA]] | 低 | DMA 控制器在设备和内存之间搬运大块数据 | 磁盘、网卡等大块传输 |

更完整的对比见 [[IO控制]]。

## I/O 软件层次

```text
用户进程
用户层IO软件
与设备无关的IO软件
设备驱动程序
中断处理程序
硬件设备
```

越靠上，越接近应用程序和统一接口；越靠下，越接近具体硬件和寄存器操作。

## 一次磁盘读的大致路径

1. 用户进程调用 `read`。
2. 标准库或运行时进入[[系统调用]]。
3. 内核检查文件描述符、权限和缓冲区。
4. 文件系统把文件偏移转换为磁盘块。
5. 与设备无关的 I/O 层检查缓存，必要时创建块设备请求。
6. [[设备驱动]]设置控制器或 [[DMA]]。
7. 设备完成后触发[[中断]]。
8. 内核唤醒等待进程，数据返回用户缓冲区。

## 常见设备

### [[磁盘]]

磁盘是典型的块设备。影响磁盘性能的主要因素包括寻道时间、旋转延迟和传输时间。

### 时钟

时钟用于维护系统时间、触发调度、统计 CPU 使用时间和处理超时。

### 监视定时器

监视定时器用于检测系统或设备是否卡死。正常运行的系统会周期性重置定时器；如果长时间没有重置，硬件可以触发恢复动作。

### 网卡

网卡负责网络帧的收发，通常配合中断、DMA 和环形缓冲区工作。每块网卡有 MAC 地址，用于局域网中的链路层识别。

### 显示器

显示器通常通过显卡和帧缓冲区工作。现代系统中，显示输出往往涉及图形驱动、显存和窗口系统。
