# 程序控制I/O

上级：[[输入输出(IO)]]

程序控制 I/O 是最简单的 I/O 方式。CPU 直接控制设备，并不断检查设备状态，直到设备准备好。

## 基本过程

1. CPU 向设备控制器发出命令。
2. CPU 反复读取状态寄存器，等待设备就绪。
3. 设备就绪后，CPU 读写数据寄存器。
4. 重复上述过程，直到数据传输完成。

## 轮询与忙等待

轮询是指 CPU 反复查询设备状态。忙等待是指 CPU 在等待期间不能做有用工作，只能不断检查条件。

这种方式实现简单，但会浪费 CPU 时间。

## 优点

- 实现简单。
- 不需要复杂中断机制。
- 对极简单或极快的设备可能够用。

## 缺点

- CPU 参与程度高。
- CPU 和 I/O 设备只能串行工作。
- 不适合大块数据传输。
