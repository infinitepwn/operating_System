# I/O软件层次

上级：[[输入输出(IO)]]

I/O 软件通常分层组织。分层的目标是把硬件细节隐藏在底层，让上层使用统一接口。

## 用户层 I/O 软件

用户层 I/O 软件包括标准库函数、格式化输入输出和[[SPOOLing]] 等。

例如 `printf` 先在用户态完成格式化，再通过系统调用进入内核写设备或文件。

## 设备独立性 I/O 软件

设备独立性也称设备无关性，意思是应用程序请求的是逻辑设备，而不是某个固定的物理设备。

在应用程序中，程序使用逻辑设备名发出请求；系统执行时，再把逻辑设备名映射到具体物理设备。

这样做有两个好处：

- 增加设备分配的灵活性。
- 易于实现 I/O 重定向，也就是改变实际输入输出设备而不修改应用程序。

设备独立性软件主要做两类工作：

- 执行所有设备共有的操作：设备分配与回收、逻辑设备名到物理设备名的映射、保护、缓冲、差错控制、统一逻辑块大小。
- 向用户层或文件层提供统一接口，例如 `read`、`write`、`open`、`close`。

更详细的内容见 [[与设备无关的IO软件]]。

## 设备驱动程序

设备驱动程序负责具体设备控制，

设置控制器寄存器、启动 I/O、处理中断结果。

驱动屏蔽硬件差异，是操作系统支持多种设备的关键。

根据不同的设备，把上层发来的read、write等抽象命令变成设备控制器的信号

## 中断处理程序

中断处理程序在设备完成工作或发生错误时运行。它通常完成最紧急的状态读取和唤醒等待进程，把复杂处理交给驱动后续执行。
