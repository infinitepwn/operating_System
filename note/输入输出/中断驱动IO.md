# 中断驱动I/O

上级：[[输入输出(IO)]]

中断驱动 I/O 让设备在完成操作后主动通知 CPU。CPU 发出 I/O 请求后，可以去执行其他进程，不必一直轮询设备。

## 基本过程

1. 进程发起 I/O 请求，进入内核。
2. 设备驱动设置设备控制器，启动 I/O。
3. 请求进程阻塞，CPU 调度其他进程运行。
4. 设备完成后触发中断。
5. 中断处理程序读取设备状态，唤醒等待进程。
6. 操作系统在合适时机重新调度该进程。

## 优点

- CPU 不需要忙等设备完成。
- 适合低速设备和异步事件。
- 能提高 CPU 与 I/O 设备的并行程度。

## 缺点

- 每次中断都有上下文保存、模式切换和处理开销。
- 高频小数据量中断可能让 CPU 花太多时间处理中断。
- 大块数据传输时，仍可能需要 CPU 参与搬运数据。

因此，大块数据传输通常结合[[DMA]]。
