# 用户层I/O软件

上级：[[输入输出(IO)]]

用户层 I/O 软件运行在用户态，通常由标准库、运行时系统和后台服务组成。它不直接控制硬件，而是通过系统调用请求内核完成 I/O。

## 标准库 I/O

例如 C 标准库中的 `printf`、`scanf`、`fread`、`fwrite` 等函数，会在用户态完成格式化、缓冲和参数处理，再通过[[系统调用]]进入内核。

标准库缓冲可以减少系统调用次数，提高性能。

## 格式化输入输出

格式化 I/O 把程序中的数据结构转换为文本，或把文本解析为程序变量。

例如 `printf("%d", x)` 会把整数转换成字符序列，再写入文件或终端。

## [[SPOOLing|假脱机系统]]

假脱机系统把对独占设备的请求先写入磁盘队列，再由后台进程按顺序提交给设备。

这样多个进程可以“同时提交任务”，但真实设备仍按顺序处理。
