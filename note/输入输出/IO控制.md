# IO控制

上级：[[输入输出(IO)]]

I/O 控制方式描述 CPU、设备控制器、内存之间如何配合完成一次输入输出。

## 三种方式

| 方式 | CPU 参与程度 | 是否忙等 | 数据搬运者 | 典型场景 |
| --- | --- | --- | --- | --- |
| [[程序控制IO]] | 高 | 是 | CPU | 简单设备、小数据量 |
| [[中断驱动IO]] | 中 | 否 | CPU 或设备控制器 | 键盘、串口、低速异步设备 |
| [[DMA]] | 低 | 否 | DMA 控制器 | 磁盘、网卡等大块数据 |

## 演进逻辑

程序控制 I/O 的问题是 CPU 需要一直等待设备；中断驱动 I/O 让 CPU 在等待期间运行其他进程；DMA 进一步减少 CPU 搬运数据的开销。

```text
程序控制I/O -> 中断驱动I/O -> DMA
CPU占用：高 -> 中 -> 低
并行程度：低 -> 中 -> 高
```

## 易混点

- 中断驱动 I/O 解决的是“CPU 不必一直等设备完成”。
- DMA 解决的是“CPU 不必亲自搬运大块数据”。
- DMA 完成后通常仍然需要中断通知 CPU。
