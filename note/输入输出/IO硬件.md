# I/O硬件

上级：[[输入输出(IO)]]

I/O 硬件包括设备本身、设备控制器、总线、设备寄存器和用于通知 CPU 的中断机制。

## 设备控制器(I/O接口)

![image.png](https://raw.githubusercontent.com/infinitepwn/note_picbed/main/20260623224015408.png)

设备通常不直接和 CPU 交互，而是通过设备控制器交互。

- 设备：真实完成输入或输出的硬件，例如磁盘、键盘、网卡。
- 控制器：管理设备细节，向 CPU 暴露寄存器和命令接口。
- 驱动程序：操作系统中知道如何控制具体控制器的软件。
设备控制器和CPU的接口用来实现CPU和设备控制器之间的通信
I/O逻辑用来控制设备
## 设备寄存器(I/O端口)

控制器通常提供几类寄存器：

- 状态寄存器：记录设备是否就绪、是否出错。
- 控制寄存器：CPU 写入命令，控制设备执行操作。
- 数据寄存器：CPU 与设备交换数据。

驱动程序通过读写这些寄存器控制设备。

## 设备控制流程

一次简单的设备控制通常包括：

1. 驱动检查设备状态寄存器。
2. 驱动把命令写入控制寄存器。
3. 设备控制器执行操作。
4. 控制器更新状态寄存器。
5. 控制器通过轮询结果、中断或 DMA 完成通知。

## I/O 编址

### I/O 独立编址

I/O 端口地址和内存地址分离，CPU 使用专门的 I/O 指令访问端口。
![image.png](https://raw.githubusercontent.com/infinitepwn/note_picbed/main/20260623224610884.png)

### 内存映像 I/O

设备寄存器被映射到内存地址空间。CPU 使用普通读写内存指令访问设备寄存器。

根据地址范围区分主存单元和I/O端口

这种方式让设备访问更统一，但必须小心缓存和访问顺序问题，因为设备寄存器不是普通内存。

## 内存映像 I/O 的注意点

- 设备寄存器访问不能像普通内存一样随意缓存。
- 编译器和 CPU 的乱序执行可能影响设备访问顺序。
- 驱动通常需要使用特定的内存屏障或非缓存映射。

## 总线

总线连接 CPU、内存和 I/O 控制器，用于传输地址、数据和控制信号。

I/O 性能不仅取决于设备速度，也受总线带宽、控制器能力和中断开销影响。
