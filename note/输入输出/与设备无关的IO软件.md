# 与设备无关的I/O软件

上级：[[输入输出(IO)]]

与设备无关的 I/O 软件位于设备驱动之上，负责提供统一接口，并处理多个设备共有的问题。

## 主要功能

- 统一命名：用路径名、设备文件或句柄标识设备。
- 统一接口：把设备访问抽象为 open、read、write、close 等操作。
- 保护：检查进程是否有权限访问设备。
- 缓冲：平衡 CPU、内存和设备之间的速度差异。
- 错误处理：把硬件错误转换为上层可理解的错误码。
- 块大小转换：把用户请求拆分或合并成设备支持的块大小。
这一层会执行系统调用

## 逻辑设备和物理设备

- 逻辑设备：应用程序看到的抽象设备名或文件名。
- 物理设备：系统实际选择的硬件设备。

逻辑设备和物理设备分离后，操作系统可以实现 I/O 重定向。例如程序写标准输出，不关心输出最终去终端、文件还是管道。

## 缓冲

缓冲区用于暂存数据，常见作用包括：

- 让高速 CPU 不必一直等待低速设备。
- 把许多小请求合并成较大请求。
- 在设备和应用程序速度不匹配时做平滑。
