# 设备驱动

上级：[[输入输出(IO)]]

设备驱动是操作系统中直接控制具体硬件设备的软件模块。它把统一的 I/O 请求转换为设备控制器能理解的命令。

## 职责

- 初始化设备。
- 接收上层 I/O 请求。
- 检查参数并转换为设备命令。
- 设置设备寄存器或 DMA 描述符。
- 启动 I/O 操作。
- 处理中断结果和错误。
- 向上层报告完成状态。

## 为什么需要驱动

不同设备的控制寄存器、命令格式、中断语义和错误处理方式都不同。操作系统不能把这些差异暴露给应用程序。

驱动负责屏蔽硬件差异，让上层通过统一接口访问设备。

## 驱动和内核结构

在[[宏内核]]中，设备驱动通常运行在内核态，效率高，但驱动错误可能影响整个系统。

在[[微内核]]中，许多驱动可以放在用户态，隔离性更好，但通信和上下文切换开销更高。
