# 无存储抽象

上级：[[内存管理]]

无存储抽象指程序直接使用物理地址，程序中的地址就是内存中的真实地址。

## 特点

- 实现简单，硬件和操作系统负担小。
- 程序直接暴露在物理内存上，缺少保护。
- 多个程序同时运行时容易互相覆盖。

## 运行多个程序的问题

如果多个程序都使用绝对物理地址，操作系统需要解决两个问题：

- 保护：一个程序不能随意访问另一个程序的内存。
- 重定位：程序装入不同物理位置时，地址需要调整。

## 常见补救

- 静态重定位：装入程序时修改地址。
- 使用基址寄存器和界限寄存器：运行时完成地址转换和越界检查。
- 使用 MPU 等硬件保护机制：给内存区域设置访问权限。
