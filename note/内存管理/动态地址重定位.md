# 动态地址重定位

上级：[[内存管理]]

动态地址重定位是在程序运行时完成地址转换。程序使用逻辑地址，硬件在每次访存时把逻辑地址转换成物理地址。

是在程序的执行过程发生的！
系统只有一个重定位寄存器
## 与静态重定位的区别

静态重定位在程序装入内存时一次性修改地址；动态重定位不修改程序本身，而是在运行时转换地址。

动态重定位的优势是进程可以在运行期间被移动，例如配合[[交换]]把进程换出后再换入到不同位置。

## 基址加界限

最简单的动态重定位使用两个寄存器：

- 基址寄存器：保存进程所在物理内存的起始地址。
- 界限寄存器：保存进程地址空间的大小。

访问流程：

1. CPU 产生逻辑地址。
2. 硬件检查逻辑地址是否小于界限寄存器。
3. 如果合法，就加上基址寄存器得到物理地址。
4. 如果越界，就触发异常，交给操作系统处理。

## 局限

- 一个进程必须占用一段连续物理内存。
- 进程增长时可能需要移动到更大的空闲区。
- 容易产生外部碎片。

这些问题推动了分页、分段和[[虚拟内存]]等机制的发展。
