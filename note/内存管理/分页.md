# 分页

上级：[[内存管理]]

把主存和进程划分为大小相等的页框和页，

主存划分成块和页框是两种划分！
分页把虚拟地址空间切成固定大小的页，把物理内存切成同样大小的页框。

- 虚拟页：虚拟地址空间中的页。
- 页框：物理内存中的页大小区域。
- 页表：记录虚拟页到物理页框的映射关系。

分页的关键好处是进程不需要占用连续物理内存。一个进程的不同页可以分散放在不同页框中。

### 页表项
![image.png](https://raw.githubusercontent.com/infinitepwn/note_picbed/main/20260623011736342.png)
• 页框号：页映射的目的

• “在/不在”位：决定缺页中断

• 保护：决定读写

• 修改（modified）位：记录页面是否修改（决定换回硬盘时是否写回）

• 访问（referenced）位：记录当页面是否正在访问

页表就是0，1，2，...页号对应页表项
## 地址转换

一个虚拟地址通常可以拆成两部分：

```text
虚拟地址 = 虚拟页号 + 页内偏移
```

转换过程：

1. 根据虚拟页号查询页表。
2. 如果页在内存中，得到对应的物理页框号。
3. 物理页框号和页内偏移组合成物理地址。
4. 如果页不在内存中，触发缺页异常。

页内偏移不需要改变，因为页和页框大小相同。

## TLB

TLB（Translation Lookaside Buffer）是地址转换缓存，用来保存最近使用过的页表项。

如果每次访存都先查页表，再访问真实数据，一次内存访问可能变成两次甚至更多次内存访问。TLB 缓存页表项后，常见页的地址转换可以直接在硬件缓存中完成。

保存在MMU中
## 缺页异常

缺页异常发生时，说明进程访问的页当前不在物理内存中。

处理流程：

1. CPU 触发缺页异常，进入内核。
2. 操作系统检查访问是否合法。
3. 如果合法，从磁盘把目标页读入空闲页框。
4. 如果没有空闲页框，执行[[页面置换算法]]选择一个页换出。
5. 更新页表，重新执行导致缺页的指令。

如果访问不合法，例如访问未分配地址，操作系统会终止进程或发送异常信号。

## 优点

- 隔离进程地址空间，增强保护。
- 程序不需要关心自己装入物理内存的哪个位置。
- 允许程序地址空间大于实际物理内存。
- 支持按需调页，提高内存利用率。
