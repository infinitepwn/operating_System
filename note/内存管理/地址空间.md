# 地址空间

上级：[[内存管理]]

地址空间是一个进程能够使用的地址集合。引入地址空间后，进程看到的是自己的逻辑地址，而不是机器上的真实物理地址。

## 为什么需要地址空间

没有地址空间时，程序直接使用物理地址，会带来几个问题：

- 多个程序可能访问同一块物理内存，互相覆盖。
- 程序必须知道自己被装入内存的具体位置。
- 操作系统很难移动进程，也很难隔离不同进程。

地址空间把每个进程都包装成“好像独占一段连续内存”的样子。

## 逻辑地址与物理地址

- 逻辑地址：程序发出的地址，也叫虚拟地址。
- 物理地址：内存硬件实际访问的地址。

地址转换由硬件和操作系统配合完成。硬件通常负责快速转换，操作系统负责维护转换所需的数据结构。

## 基址寄存器和界限寄存器

基址寄存器记录进程在物理内存中的起始位置，界限寄存器记录进程可用地址空间的大小。

```text
物理地址 = 基址寄存器 + 逻辑地址
```

每次访存时，硬件先检查逻辑地址是否超过界限。如果越界，就触发异常，进入操作系统处理。

## 作用

- 重定位：同一个程序可以装入不同物理位置运行。
- 保护：进程不能访问自己地址空间之外的内存。
- 抽象：程序可以从地址 0 开始编写，不必关心物理内存布局。
