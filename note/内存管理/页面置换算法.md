# 页面置换算法

上级：[[虚拟内存]]

页面置换算法用于在物理页框不足时选择一个页换出，为即将调入的页腾出空间。

## 触发场景

1. 进程访问某个虚拟页。
2. 页表显示该页不在物理内存中。
3. CPU 触发缺页异常。
4. 操作系统需要从外存调入该页。
5. 如果没有空闲页框，就必须选择一个页换出。

## 最优置换算法

最优算法总是换出未来最长时间不会再被访问的页。

它的缺页率最低，但实际系统无法提前知道未来访问序列，所以主要用于理论比较。

## FIFO

FIFO（First In First Out）换出最早进入内存的页。

优点是实现简单；缺点是可能换出经常使用的页，并且可能出现 Belady 异常：分配更多页框后缺页次数反而增加。

## LRU

LRU（Least Recently Used）换出最长时间没有被访问的页。

它利用时间局部性：最近被访问的页，接下来仍可能被访问。

真实 LRU 需要精确记录访问顺序，开销较高，因此实际系统常用近似算法。

## Clock

Clock 算法是 LRU 的近似实现。每个页维护一个访问位，所有页框按环形队列组织。

扫描时：

1. 如果访问位为 0，选择该页换出。
2. 如果访问位为 1，把它清 0，并跳过该页。
3. 指针继续向后移动，直到找到可换出的页。

Clock 算法实现简单，效果通常比 FIFO 好。

## 抖动

如果进程可用页框太少，系统会频繁发生缺页，不断在内存和磁盘之间换页，CPU 大量时间浪费在等待 I/O 上，这种现象称为抖动。

解决思路包括减少并发进程数量，或为进程分配足够的页框，使其工作集能留在内存中。
